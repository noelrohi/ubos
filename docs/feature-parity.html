ubos parity map

A native Swift macOS plan for a personal OpenUsage-style menu bar app. Scope is intentionally smaller: Claude Code, Codex, OpenCode Go, and Cursor first.

OpenUsage v0.6.26Latest GitHub release inspected.
Tauri stackJavaScript plugins, TypeScript UI, Rust host.
Native targetSwiftUI + AppKit menu bar app, no webview needed.
Codex providerOpenUsage uses Codex CLI OAuth and ChatGPT usage endpoints.
Claude Code parity
  • Fetch live limits from Anthropic OAuth usage: 5h session, weekly, Sonnet, Claude Design, and extra usage.
  • Read local transcripts from ~/.claude/projects/**/*.jsonl with ccusage-style dedup, pricing, and 5h block fallback.
  • Show today, yesterday, and 30-day cost/token text rows; costs are API-equivalent estimates.
Cursor parity
  • Read desktop SQLite auth from ~/Library/Application Support/Cursor/User/globalStorage/state.vscdb.
  • Fallback to Keychain services cursor-access-token and cursor-refresh-token.
  • Fetch current period usage, plan info, credit grants, Stripe balance, and request fallback.
OpenCode Go parity
  • Read local observed spend from ~/.local/share/opencode/opencode.db.
  • Detect auth at ~/.local/share/opencode/auth.json.
  • Show 5h, weekly, and monthly spend against fixed Go limits.
Codex parity
  • Read Codex CLI OAuth credentials from file or Keychain.
  • Fetch session, weekly, review, and credits usage from chatgpt.com/backend-api/wham/usage.
  • Use local Codex logs later for token trend/detail metrics.

Manual refresh User can retry failed providers and refresh usage. Refresh button in popover and per-provider retry on error. Must match SwiftUI button in popover. Disable while refresh is running.

Auto update OpenUsage releases auto-update. Not required for personal-only app, but easy to add before sharing. Defer Use Sparkle only if distributing outside your machine.

Recommended Build Order

Step Deliverable Why first
1 Native menu bar shell with mock provider snapshots. Locks in the macOS UX before network and auth complexity.
2 OpenCode Go and Claude Code local adapters. Fastest real providers because they are local file/SQLite only.
3 Cursor adapter. Highest-value parity provider, but includes token refresh and fallbacks.
4 Codex adapter. Uses OpenUsage-documented OAuth and usage endpoint behavior.
5 Settings, shortcut, local API, polish. Add after data correctness is proven.

This plan intentionally avoids the OpenUsage plugin runtime for now. For a personal Swift app with three providers, native adapters are smaller, easier to debug, and fit macOS sandbox/keychain behavior better.
